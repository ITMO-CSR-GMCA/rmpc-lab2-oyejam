## 1. Introduction

This project implements kinematic modeling, workspace analysis, inverse kinematics, trajectory planning, and inverse dynamics of a UR5 industrial manipulator using the Robotics Toolbox for Python.

The objective is to connect theoretical robot motion planning with numerical simulation of a 6-DOF robotic system.

## 2. Robot Model and Kinematics

The UR5 robot is defined using standard DH parameters:

\[
T_i^{i+1} =
\begin{bmatrix}
\cos\theta_i & -\sin\theta_i\cos\alpha_i & \sin\theta_i\sin\alpha_i & a_i\cos\theta_i \\
\sin\theta_i & \cos\theta_i\cos\alpha_i  & -\cos\theta_i\sin\alpha_i & a_i\sin\theta_i \\
0 & \sin\alpha_i & \cos\alpha_i & d_i \\
0 & 0 & 0 & 1
\end{bmatrix}
\]

The forward kinematics is:

\[
T_0^n(q) = \prod_{i=1}^{n} T_i^{i+1}(q_i)
\]

---

## 3. Workspace and Trajectory Planning

Workspace is generated by sampling joint limits:

\[
q_i \in [q_{min}, q_{max}]
\]

End-effector position:

\[
p = FK(q)
\]

Trajectory interpolation uses cubic polynomial:

\[
q(t) = 3s^2 - 2s^3, \quad s = \frac{t}{T}
\]

---

## 4. Inverse Dynamics (Newton–Euler)

Robot dynamics is expressed as:

\[
\tau = M(q)\ddot{q} + C(q,\dot{q})\dot{q} + G(q)
\]

### Special cases:

- Dynamic: \( \dot{q} \neq 0, \ddot{q} \neq 0 \)
- Quasi-static: \( \ddot{q} \approx 0 \)
- Static: \( \dot{q} = 0, \ddot{q} = 0 \)

## 5. Discussion and Conclusion

The UR5 model successfully demonstrates:

- Forward and inverse kinematics
- Workspace generation
- Smooth trajectory planning
- Dynamic torque computation

### Key limitation:

UR5 toolbox model uses simplified dynamics (partial inertia, friction, and actuator data), therefore results are best interpreted as **theoretical validation rather than full physical simulation**.
